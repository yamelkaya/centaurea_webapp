This is a series of articles about different aspects of customization of Dynamics CRM and Salesforce. In my previous post I’ve already described security model in both systems and how it can be modified for particular busniness purposes. So, today I want to say what kind of customization supports both systems concerning UI and business model. I will try to pay attention to such questions as:

  1. Business model configuration: changes in existing objects/fields and creating new
  2. Page/Form layout configuration: particular object appearance to the different users
  3. Homepage layout configuration: dashboard components and their layout
  4. Workflows configuration

Business model configuration

So, all CRM systems have base range of standard objects that participate in sales process and you can operate with them (most common are Account, Contact, Opportunity etc.) But very often we face with the situation when in some company these entities are actively used but vary from the standard and require minor or significant changes. Both systems supports these kind of changes with some limitations.

Feature Dynamics CRM Salesforce
Rename standard objects Yes* Yes*
Rename standard fields Yes* Yes*
*- some objects or fields can not be renamed directly (sometimes it can be done through the database or custom code)

The second thing we want to do is create new object (with custom fields and relationships). This is a base feature that present in both. You can add new object, configure new fields for them, set up relationship with standard and new objects. But Salesforce definitely do it better. Well, here's the deal. It is very common request from client to add multiple picklist, autonumber or rich text area and it’s a pleasure to know that Salesforce supports them by default. I don’t want to say that in Dynamics CRM it is not possible to do at all but it requires more efforts from you (find and reuse existing component for that or create it from scratch). Of course, it is more painful especially when it is quite basic requirement.

Feature Dynamics CRM Salesforce
Create new object Yes Yes
Add new field for standard object Yes* Yes*
Add new field to new object Yes Yes
Add new/standard object to Site Map/Tabs Yes Yes
Use new object in reports Yes Yes
Many-to-many relationship Yes Yes
Many-to-one relationship Yes Yes
One-to-one No** No**
Different data types Yes*** Yes***
Calculated/roll-up fields Yes (with 2015 update) Yes
Configure object access per security role/profile Yes Yes
Configure field access per security role/profile No Yes
* - not all out-of-the-box objects supports modification
** - does not exist as standard relationship type but can be created based on many-to-one
*** - list of available types for Salesforce and Dynamics are different

Page layout configuration

For optimal information perception very often we need to rearrange it or replace with some graphical representation (e.g. person full name and contact data are important and must go higher). Below the list of configuration possibilities.

Feature Dynamics CRM Salesforce
Control fields/lists visibility Yes Yes
Control field access modifier (required, read only) Yes Yes
Customize fields appearance Yes Yes
Customize buttons Yes Yes
Add external resources (such as iframes and html web pages) Yes Yes
Add charts and reports Yes Yes
Configure multiple form layouts Yes Yes
Configure page layouts per security role/profile Yes Yes

I haven't noticed any significant differences. You can achieve the same results in both CRMs.

Homepage configuration

Every time when user comes to system he will see the homepage (dashboard). That’s why it is important to show him the most actual information on it (e.g. his open activities, or hot opportunities). When person have a lot of responsibilities it requires multiple dashboards in order to show key indicators for each area. Dashboard is a good solution for that.

Feature Dynamics CRM Salesforce
Add lists/charts/reports Yes Yes
Add external resources (such as iframes and html web pages) Yes Yes
Add links Yes Yes
Configure multiple dashboards Yes Yes
Configure dashboards per security role/profile Yes Yes

Workflows configuration

Except of standard objects CRM systems have some workflows which covers basic scenario of work but it can be extended or even disabled or replaced with developers forces. I don’t want to describe facilities which used for that because this is topic for separate discussion. The only thing I’d like to add is that both systems are flexible enough for implementation of workflows of different levels of complexity.

Conclusion

So, when the question is related to UI customization I want to say that Salesforce will provide more comprehensive set of out-of-the-box features for mapping your business model to CRM system. Some of positive moments about Dynamics CRM is that custom developments fully based on standard stack of technologies (javascript, .NET, SSRS) and it does not require learning something new (like Apex). But anyway (although Dynamics CRM team do a lot in system improvement), for now Salesforce provides better experience.
